# Requirement

We are set out to build the framework for a company in the age of agentic LLM from the ground up. The analogy is the removal of the "line shaft system" when transitioning from the Steam to Electricity.

These are the key question to answer:
1. If we start a new company today from nothing, how do we run the company in a way that better levarage the power of AI agents?
2. What would be the difference between any established, conventional company, to what we are trying to design, in leverage AI agents to acceelrate company growth?

# Implementation

In today's day and age, AI agents is utilised more like a tool in a toolbox, where it is used by expert in the field to complete a task(s). This is because the operation of AI agents, with LLM at its core, require the appropriate input of data, for it to produce valuable outcome with enormous leverage. The typical saying of Garbage In, Garbage Out (GIGO) is very much true in this new revolution.

Thus, we have identified a gap where a convetional companies' data, or the information it represent, is not well curated and structured to welcome the true power of LLM based AI agents.

With that, we introduce the concept of RIVERS, it is similar to a "programming language" of a company. RIVERS represent the following:

## R - Requirements

What needs to be done, something that will take effort from any person in a company

## I - Implementation

What has been done to address the requirements

## V - Verification

Method and outcome when judging how the implementation address the requirements, with evidence.

## E - Educate

What do we learn? Learning from either the implementation process AND/OR outcome can be educational

## R - Retry

If we need to retry what is done in this RIVERS, what would we do differently

## S - Standardize

From this RIVERS, what can we set as dafult going forward, the default can still be challenged with another RIVERS.

# Verification

Notice how the conception of RIVERS is performed by using RIVERS, this very first RIVERS will be the evidence on what we are set out to do, and allow later human or AI agents to continue the development of RIVERS.

AI Agents is a core partner in the founding of RIVERS, the concept has been polished through communication with LLM to ensure RIVERS is general, easy to use, and ready for te future.

# Educate

From the process of conceptualizing RIVERS, one thing stood out to me, it does not need to be perfect on the get go, this RIVERS that born the concept of RIVERS will look miniature to what RIVERS will grow and look like in the future.

Throughout this process of starting RIVERS, I have so many question in my mind:

"What is the perfect RIVERS title structure?"
"How do we store RIVERS?"
"Do we use existing platform like GitHub?"
"Should we use Education instead of Educate?"

In hindsight, RIVERS will help to resolve this perfectionist mentality, all of these can be answered in a later RIVERS if the need arises, and a track of evidence can be studied to understand why RIVERS has become what it became.

# Standardization

RIVERS will be an open-source standard for anyone to practice, in their personal live, in an organization, in a government, and more that we couldn't come up with now.

A GitHub repository will be the first home where RIVERS is defined.
